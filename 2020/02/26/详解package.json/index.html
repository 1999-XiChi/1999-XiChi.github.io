详解package.json

package.json文件是一个 JSON 文件，位于项目的根目录中。它包含关于项目的重要信息，包括可读元数据（如项目名称和说明）以及功能元数据（如程序包版本号和程序所需的依赖项列表）。

1.常见字段

name

如果将软件包发布到 NPM，则 name 属性是必需的，并且必须是唯一的。

version

如果将软件包发布到 NPM，确保version字段保持更新。

语义版本控制的概念很简单：所有版本都有3位数字：x.y.z。

  • 第一位是主要版本
  • 第二个数字是次要版本
  • 第三位数是补丁版本

使用^ ~

  • ^：如果您^0.13.0在运行时编写该代码，npm update则可以更新到补丁程序和次要发行版：0.13.1，0.14.0等等。
  • ~：如果您编写~0.13.0，则在运行npm update时可以更新至补丁程序发行版：0.13.1可以，但0.14.0不是。

license

如果你不想提供许可证，或者明确不想授予使用私有或未发布的软件包的权限，则可以将 UNLICENSED 作为许可证

  • MIT只要用户在项目副本中包含了版权声明和许可声明，他们就可以拿你的代码做任何想做的事情，你也无需承担任何责任。
  • GPL这是一种copyleft许可证，要求修改项目代码的用户再次分发源码或二进制代码时，必须公布他的相关修改。V3版本与V2类似，但其进一步约束了在某些限制软件更改的硬件上的使用范围。
